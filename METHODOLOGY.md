# Методология и ограничения

## Источник данных

Негативные отзывы берутся из публичного Steam Reviews API:

```
https://store.steampowered.com/appreviews/{app_id}?json=1
```

Это тот же источник, который показывает отзывы на странице магазина Steam.
Ключ API не нужен. Ответ приходит в JSON. Проект запрашивает не все отзывы, а
сразу только негативные.

Основные параметры запроса, которые я использую:

- `filter=recent` — отзывы отсортированы по дате создания. Это позволяет
  постранично дойти до конца выборки (при `filter=all` API отдаёт отзывы
  бесконечно и никогда не заканчивается).
- `review_type=negative` — Steam отдаёт только отзывы, где игрок нажал
  «Не рекомендую».
- `language=russian` и `language=english` — беру два языка отдельными проходами.
- `num_per_page=100` — максимум отзывов за один запрос.
- `cursor` — курсор для перехода на следующую страницу. Первый запрос идёт с `*`,
  дальше подставляется значение из ответа.
- `purchase_type=all` — учитываю все отзывы, а не только купленные в Steam.

## Как отбираются негативные отзывы

У каждого отзыва есть поле `voted_up`:

- `True` — игрок рекомендует игру.
- `False` — игрок не рекомендует игру.

Основной анализ проекта строится только по отзывам `voted_up = False`. Эти
отзывы уже отбираются на этапе запроса через `review_type=negative`, поэтому
положительные отзывы не скачиваются и не участвуют в расчетах. Это официальная
оценка самого игрока, а не результат автоматической модели.

## Как ищутся категории жалоб

Категории жалоб ищутся по ключевым словам. Для каждой категории (читы,
античит, оптимизация, серверы, баланс, монетизация, прогрессия и т.д.) задан
расширенный набор слов на русском и английском. Отзыв относится к категории,
если содержит хотя бы одно её слово. Один отзыв может попасть сразу в несколько
категорий.

Это простой и прозрачный подход. Он не понимает смысл, а только ищет слова,
поэтому возможны ложные срабатывания. Например, фраза "no cheaters" всё равно
может попасть в категорию "читы", потому что содержит слово `cheaters`.
Зато результат легко проверить руками и расширять списки слов.

## Что считается в итоговых таблицах

- `negative_complaint_summary.csv` — сколько негативных отзывов собрано по каждой игре и какой период они покрывают.
- `negative_theme_summary.csv` — общий топ категорий жалоб.
- `negative_themes_by_game.csv` — тепловая матрица: игра × категория жалобы.
- `game_complaint_points.csv` — топ проблем отдельно по каждой игре.
- `monthly_negative_reviews.csv` — динамика негативных отзывов по месяцам.
- `monthly_top_complaint_categories.csv` — динамика топ-категорий жалоб по месяцам.

## Ограничения

- В выборку попадают только отзывы Steam. Мнения из Discord, Reddit, YouTube
  сюда не входят.
- Отзывы на русском и английском не покрывают всю аудиторию (например, большой
  пласт китайских отзывов не анализируется).
- Steam иногда фильтрует «review bombing», это может влиять на состав отзывов.
- Наигранное время (`playtime_at_review`) заполнено не у всех отзывов.
- Если `MAX_NEGATIVE_REVIEWS_PER_GAME = None`, тетрадка идёт по страницам Steam
  API до конца доступной истории. Если поставить число, будет взята свежая
  ограниченная выборка негативных отзывов.
- Ключевые слова не заменяют ручной качественный анализ, но помогают быстро
  увидеть повторяющиеся проблемы.
